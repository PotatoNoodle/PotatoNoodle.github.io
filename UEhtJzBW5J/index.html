节省成本，我的静态博客

2025-10-26 2 min read

在很多年前我就跟风搭建了一个博客（WP），后来域名到期，也没什么内容可输出，就关闭了。
接着一时兴起，用更轻量的typecho搭建了一个博客，关闭原因同上。
后来又在家里的nas上运行了一个博客，用CF的tunnel发布到互联网上，但经常被“上市”（好，把他们全部上市！），很扫兴，于是关闭了。
现在用pages搭建了静态博客，直接解决了服务器的烦恼和开销，只剩下我的域名费用了。

简单整理下思路。

搭建博客省略，照猫画虎就是。

为了方便的整理照片获得当下路径内所有文件名，并且把它加上markdown的图片语法，顺带还整了个脚本，直接形成列表并匹配markdown，留在这里方便以后使用。

import os
folder = os.getcwd()  
files = sorted(os.listdir(folder))


exts = ('.jpg', '.jpeg', '.png')
for f in files:
    if f.lower().endswith(exts):
    print(f"![]({os.path.join(folder, f)})")

哈哈，然后发现每次要同步几百兆，这可不行，而且有以下几个问题：
1.本地编辑好后，图片太大，同步上去太慢。
2.大量占用仓库空间，微软可能封我号。
3.偶发性阻断导致可能无妨正常访问。
于是我换个思路解决了图床问题。

把自己家里路由器上的webdav（网盘）当作图床，把自己需要上传的照片，放进了一个特定的共享的文件夹，再用markdown的"*![]"来显示图片，效果如下。

就是这么个思路，域名付费（应该还有5年），服务器成本0，图床自建（家里的路由器），实现了我更便宜的博客自由！
